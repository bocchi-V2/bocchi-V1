## 🟢 ปัญหาระดับเล็กน้อย (ส่งผลต่อ UX เล็กน้อย)

| # | ปัญหา | สาเหตุ | วิธีแก้ |
|---|-------|--------|---------|
| 11 | **บน iOS Safari ดาวน์โหลดไม่ลงใน Camera Roll** | iOS บังคับเปิดวิดีโอใน browser แทน | กด hold วิดีโอ → Save to Files |
| 12 | **ไฟล์ใหญ่ (>500MB) กิน RAM** | Fetch + Blob เก็บทั้งไฟล์ใน memory | บนมือถือ RAM น้อยอาจ crash — แนะนำเปิดแท็บใหม่แทน |
| 13 | **Rate Limiting** | ถ้าใช้บ่อยๆ TikWM อาจ throttle IP | รอสักครู่แล้วลองใหม่ |
| 14 | **ชื่อไฟล์มีอักขระพิเศษ** | บน Windows ชื่อไฟล์บางตัวไม่ถูกต้อง | โค้ด sanitize ชื่อแล้ว แต่อาจพลาดบางกรณี |
| 15 | **ไม่ทำงานบน HTTP** | บาง browser ปฏิเสธ mixed content | ใช้ HTTPS เสมอ (GitHub Pages ใช้ HTTPS อัตโนมัติ) |

## 📋 สรุปขีดจำกัดที่ตั้งใจให้เป็น

- ✅ **ไม่มี Backend** — ทุกอย่างทำใน browser ล้วนๆ
- ✅ **ไม่เก็บข้อมูล** — ไม่มี database, ไม่มี server log
- ❌ **ทำภาพ/วิดีโอชัดขึ้นจริงๆ ไม่ได้** — เว็บแค่ดาวน์โหลดไฟล์ที่ API ส่งมา คุณภาพขึ้นอยู่กับต้นฉบับ (แต่เลือก HD เสมอแล้ว)
- ❌ **ไม่รองรับ Premium/Restricted content** — เช่น TikTok LIVE, YouTube Premium

> 💡 **คำแนะนำ**: ถ้าต้องการความเสถียรมากขึ้น ควรทำ Backend (Node.js/Python) เพื่อเรียก TikWM API จาก server แทน browser — จะหลีกเลี่ยงปัญหา CORS ได้ 100%
